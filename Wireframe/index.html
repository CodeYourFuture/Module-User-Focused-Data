Version Control System

One of the most popular Version control systems tools in use today is called Git.

What is Git?

Git is a version control software that allows developers to create and manage different versions of a project. It is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

Why do developers need Git?

Developers need Git because it helps with source code management and collaboration. Git allows multiple developers to work on a project simultaneously. Git tracks changes to files, including who made them and when. This allows developers to manage multiple versions of a project and revert to previous versions if needed. What is a branch in Git?

A branch in Git is simply a lightweight movable pointer to snapshot of changes (commits) in the code base.
In Git, there is a special branch (default) which holds the aggregated changes. It is often name master or main. Only changes that have been agreed upon by the team is merged into the main branch. Collaborators/Developers will cut out from main to an experimental/feature branch to their development work.
