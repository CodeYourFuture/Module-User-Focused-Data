What is Git?

It is a version control software platform for developers to create and manage different versions of a project. We end up creating different versions of a project overtime through commits. A commit is sort of a snapshot of the project at a particular point in time. Commits are like particular versions of the project at particular times. Basically Git is a tool/platform for teams to store the history of different versions of their project.

Summary

Git uses commits to store: What changed in a particular commit, who created the change, what time the change happened and what was the previous commit. These changes would take place on a project and history and these projects are called repositories.

Why do Developers Need Git?

One of the biggest advantages of Git is its branching capabilities. Unlike centralized version control systems, Git branches are cheap and easy to merge. This facilitates the feature branch workflow popular with many Git users.
